# Astra ML Lane Intelligence

Local queue-depth estimation service for Astra checkout lanes. The service runs a
YOLOv8n person-detection model with ONNX Runtime and exposes both gRPC
(`LaneIntelService`) and HTTP health endpoints.

## Responsibilities

- Receive lane/store identifiers via gRPC.
- Pull the current camera frame (production integrations provide a frame
  provider; a default placeholder is included).
- Run YOLOv8n inference on CPU.
- Return queue depth, estimated wait time, and capture timestamp.

## Requirements

- Python 3.13+
- A YOLOv8n ONNX model at `models/yolov8n.onnx`

## Installation

Create a virtual environment and install the package:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

## Generate the ONNX model

A deterministic stub model is sufficient for tests and CI:

```bash
python -m models.download_model
```

To download the real YOLOv8n model from Ultralytics:

```bash
python -m models.download_model --real
```

## Generate protobuf stubs

The stubs are committed, but you can regenerate them with:

```bash
python -m grpc_tools.protoc \
    --proto_path=proto \
    --python_out=proto \
    --grpc_python_out=proto \
    proto/lane.proto proto/common.proto
```

## Run the service

Start both the gRPC server (port 50051) and the HTTP health server (port 8088):

```bash
python -m src.main
```

Or with explicit ports:

```bash
python -m src.main --port 8088 --grpc-port 50051 --model models/yolov8n.onnx
```

## HTTP endpoints

| Endpoint | Description                                      |
|----------|--------------------------------------------------|
| `GET /health` | Service health and model load status            |
| `GET /ready`  | Readiness probe; 503 until model is loaded      |
| `GET /live`   | Liveness probe; always 200 while running        |

## gRPC methods

```proto
rpc GetQueueDepth(QueueDepthRequest) returns (QueueDepthResponse);
rpc StreamQueueDepth(QueueDepthRequest) returns (stream QueueDepthResponse);
```

## Testing

Run the test suite with pytest:

```bash
python -m pytest
```

Tests gracefully skip if the stub ONNX model cannot be generated (the
`download_model.py` script is only required for model-dependent tests).

## Docker

Build and run a multi-stage image:

```bash
docker build -t astra-ml-lane-intel .
docker run -p 8088:8088 -p 50051:50051 astra-ml-lane-intel
```

## Configuration

| Environment variable | Default           | Description                           |
|----------------------|-------------------|---------------------------------------|
| `MODEL_PATH`         | `models/yolov8n.onnx` | Path to the ONNX model            |
| `HOST`               | `0.0.0.0`         | HTTP server host                      |
| `PORT`               | `8080`            | HTTP server port                      |
| `GRPC_PORT`          | `50051`           | gRPC server port                      |
| `GRPC_WORKERS`       | `10`              | gRPC thread-pool size                 |
