# Astra-Service Gateway

The API gateway for Astra-Service. It terminates kiosk-facing REST traffic,
enforces request IDs, structured logging, CORS, Redis-backed rate limiting,
JWT authentication and Prometheus metrics, then proxies requests to downstream
services via HTTP reverse proxy or gRPC clients.

## Structure

```
cmd/gateway/main.go          # entry point, config parsing, graceful shutdown
internal/config/config.go    # strict environment-variable parsing
internal/server/server.go    # Fiber v3 app and middleware chain
internal/routes/routes.go    # route registration and downstream proxies
internal/auth/jwt.go         # EdDSA primary / RS256 fallback JWT validation
internal/ratelimit/...       # Redis token-bucket rate limiter
internal/metrics/...         # Prometheus HTTP middleware
internal/health/...          # readiness checker for Postgres, Redis, NATS
internal/clients/...         # gRPC client registry for downstream services
docs/                        # OpenAPI spec generated by swag
```

## Middleware chain

```
RequestID → structured JSON Logger → Recover → strict whitelist CORS →
Redis token-bucket RateLimit → JWT EdDSA/RS256 Auth →
Prometheus Metrics → Handler
```

## Configuration

All configuration is read from environment variables. Required values must be
present; missing required values prevent startup.

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `GATEWAY_PORT` | no | `8080` | HTTP listen port |
| `ASTRA_ENV` | no | `development` | runtime environment |
| `ASTRA_LOG_LEVEL` | no | `info` | debug/info/warn/error |
| `GATEWAY_ALLOWED_ORIGINS` | no | `http://localhost:5180` | comma-separated CORS whitelist |
| `GATEWAY_RATE_LIMIT_RPS` | no | `50` | token-bucket refill rate |
| `GATEWAY_RATE_LIMIT_BURST` | no | `100` | token-bucket burst size |
| `GATEWAY_JWT_ISSUER` | no | `astra-service` | expected JWT issuer |
| `GATEWAY_JWT_AUDIENCE` | no | `astra-gateway` | expected JWT audience |
| `GATEWAY_JWT_EDDSA_PUBLIC_KEY` | * | | base64 or PEM EdDSA public key |
| `GATEWAY_JWT_EDDSA_PUBLIC_KEY_PATH` | * | | path to EdDSA public key PEM |
| `GATEWAY_JWT_RSA_PUBLIC_KEY` | * | | base64 or PEM RSA public key |
| `GATEWAY_JWT_RSA_PUBLIC_KEY_PATH` | * | | path to RSA public key PEM |
| `DATABASE_URL` | yes | | Postgres DSN |
| `REDIS_URL` | yes | | Redis URL |
| `NATS_URL` | yes | | NATS URL |
| `*_SERVICE_URL` | no | `http://localhost:8081` | downstream HTTP base URL |
| `*_SERVICE_GRPC_ADDR` | no | `localhost:5005x` | downstream gRPC address |

`*` At least one EdDSA or RSA key must be configured.

## Running locally

```bash
# from the repository root
export DATABASE_URL="postgresql://astra:astra@localhost:5432/astra?sslmode=disable"
export REDIS_URL="redis://localhost:6379/0"
export NATS_URL="nats://localhost:4222"
export GATEWAY_JWT_EDDSA_PUBLIC_KEY="$(base64 -i dev/eddsa.pub)"

cd astra-service/services/gateway
go run ./cmd/gateway
```

## Tests

```bash
cd astra-service/services/gateway
go test -race ./...
```

## OpenAPI

The OpenAPI spec is generated from swaggo comments in the handlers.

```bash
cd astra-service/services/gateway
swag init -g cmd/gateway/main.go
```

Generated files are written to `docs/` and served under `/docs/`.

## Docker

```bash
docker build -f astra-service/services/gateway/Dockerfile -t astra-gateway .
docker run -p 8080:8080 --env-file .env astra-gateway
```

## Endpoints

| Method | Path | Notes |
|--------|------|-------|
| GET | `/health` | public health |
| GET | `/live` | public liveness |
| GET | `/ready` | readiness (Postgres, Redis, NATS) |
| GET | `/metrics` | Prometheus metrics |
| GET | `/docs/*` | OpenAPI documentation |
| * | `/v1/menu/*` | proxied to menu service |
| * | `/v1/cart/*` | proxied to cart service |
| * | `/v1/order/*` | proxied to order service |
| * | `/v1/inventory/*` | proxied to inventory service |
| * | `/v1/payment/*` | proxied to payment service |
| * | `/v1/sync/*` | proxied to sync service |

All `/v1/*` routes except preflight OPTIONS require a valid JWT.
